## 2. Tracking Performance Issues

```Questions 2: How would you track down a performance issue in production? Have you ever had to do this?```

To track performance issues in production, I would:

1. Review application and server logs for errors or slow responses
2. Monitor performance with tools like New Relic, Datadog, or Blackfire
3. Identify slow database queries using MySQL slow query logging
4. Measure response time at each layer: frontend, backend, database
5. Reproduce the issue in staging if possible
6. Optimize the root cause: indexes, query tuning, caching, or code refactor
7. Validate the fix using monitoring and load testing

I have handled production performance issues before, mainly by identifying slow SQL queries and optimizing database access.
